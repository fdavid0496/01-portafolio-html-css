Proyectos académicos

Proyectos donde demuestro habilidades técnicas y de trabajo en equipo: UI en Angular, APIs REST con Spring Boot y prácticas de desarrollo ordenado.

  • Aplicación Front-End para gestión de reportes financieros

    Contexto universitario (Vicerrectoría Financiera) · Integración con Back-End en Spring Boot

    Rol: Desarrollador Front-End. Implementé vistas e interfaces de usuario enfocadas en la gestión de reportes, priorizando orden, consistencia y una estructura mantenible.

    Aportes clave: construcción de componentes, routing, pipes, consumo de datos desde el Back-End y aplicación de TypeScript de forma obligatoria para mejorar calidad y legibilidad.

    • Tecnologías: Angular · TypeScript
    • Integración: API Back-End (Spring Boot)
    • Metodología: SCRUM
    • Enfoque: Arquitectura por capas

    Entregables

    • Interfaz modular basada en componentes.
    • Navegación y estructura de rutas organizada.
    • Pantallas orientadas a gestión y visualización de reportes.

  • API REST para gestión de publicaciones intelectuales

    Gestión de publicaciones de docentes · Persistencia en MySQL · Enfoque en capas

    Rol: Desarrollador Back-End. Implementé la lógica principal para gestionar publicaciones, aplicando arquitectura por capas y una API REST clara.

    Aportes clave: capa de servicio y repositorio, DTOs, mappers, manejo de excepciones (incluyendo personalizadas) y estructura pensada para crecer.

    • Tecnologías: Spring Boot · MySQL
    • Arquitectura: REST · Capas · DTOs
    • Metodología: SCRUM
    • Calidad: Excepciones · Mappers

    Entregables

    • Endpoints REST para operaciones de gestión.
    • Estructura por capas para facilitar mantenimiento.
    • Validación y control de errores con excepciones.

Proyectos sociales (impacto institucional)

Iniciativas donde el foco está en el servicio y la trazabilidad para usuarios: transparencia, seguimiento y claridad en el proceso.

  • Sistema web de trazabilidad PQRSF (Universidad del Cauca)

    Seguimiento de solicitudes por etapas · Enfoque en requisitos y validación con cliente

    Rol: Analista. Mi función fue interactuar con el cliente para levantar, documentar y validar requisitos funcionales y no funcionales, buscando que el producto fuera claro para el usuario final.

    Problema que resolvía: permitir que el usuario final consultara la trazabilidad de su solicitud (por ejemplo, una queja) y entendiera en qué etapa se encontraba, reduciendo incertidumbre y mejorando transparencia.

    • Disciplina: Ingeniería de Requisitos
    • Entregables: Épicas · HU · Criterios · Mockups
    • Metodología: KANBAN
    • Habilidad: Validación y verificación

    Lo que entregué / fortalecí

    • Backlog orientado a valor: épicas e historias trazables.
    • Criterios de aceptación para reducir ambigüedad.
    • Mockups para alinear expectativas antes de construir.
    • Ciclos de revisión con cliente (V&V) para asegurar dirección correcta.

Proyectos de negocio (y proyección)

Experiencias con contexto más cercano a operación real (gestión de contratos) y el tipo de retos donde busco crecer como desarrollador.

  • Sistema web para gestión de contratos (Unidad de Salud)

    División TIC (Universidad del Cauca) · Requisitos con enfoque profesional y validación continua

    Rol: Ingeniero de Requisitos. Realicé el levantamiento de requisitos funcionales y no funcionales y acompañé la validación constante con el cliente para mantener el alcance controlado.

    Aportes clave: elaboración de épicas, historias de usuario, criterios de aceptación, mockups y herramientas de organización como story mapping y formato Merlin (entre otros insumos de trabajo).

    • Entregables: HU · Mockups · Story Mapping
    • Metodología: SCRUM
    • Habilidad: Comunicación con cliente

    Lo que aporté al equipo

    • Requisitos claros para reducir reprocesos.
    • Validación frecuente para asegurar alineación y avance.
    • Estructura de entregables lista para desarrollo incremental.
